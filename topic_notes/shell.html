Shell

Snippets of shell code that I most commonly use.


pwd - print working directory

head hello.py - shows first 10 lines of the hello.py file

ls - list contents of current directory

ls ~ - list contents of home directory

ls Desktop - list contents of Desktop directory (a folder in the current directory)

cd - change directory e.g. cd Desktop/

ls / - list contents of root directory

ls -1 - list contents of current directory, one file/folder per line. Note this is a 1 not an l

ls -lh - list contents of current directory, including details, and show numbers in human readable format (e.g. 40K). Note this is an l not a 1.

ls t* - list contents of current directory where the filename starts with t (case sensitive)

ls | grep ^t - list contents of current directory where the filename starts with t (case sensitive)

ls | grep -i ^t - list contents of current directory where the filename starts with t (ignores case)

ls | grep -i ^[a-g] - list contents of current directory where the filename starts with any letter between a and g inclusive (ignores case)

ls | grep -i ^t.*do.* - list contents of current directory where the filename starts with t and contains the word 'do' (ignores case). Note .* means any number of any type of character. ^ means start of string.

ls | grep -i \.py$ - list contents of current directory where the filename ends with .py and ignore case (note that the back slash before the . means the . is treated literally rather than as a special command. $ means end of string)

ls | grep -i ^m.*_.*\.txt$ - list contents of current directory where the filename starts with m, contains an underscore, and ends with .txt (ignoring case)

history - list previous commands

!125 - re-run command at line 125 in the history

clear - clear the shell window by scrolling down

CTRL-D - exit

touch - creates an empty file, or if used with an existing file, sets the latest accessed time to the current time

cp - copy

mv - rename

rm - delete file

rmdir - remove EMPTY folder

rm -r - remove folder and all sub-contents

chmod u+x hello.py - allow hello.py to be executed

ps -ef - list all processes running on the system

ps -ef | less - list all processes running on the system but only show what fits on the screen, then let me scroll through the rest

ps -ef | head - list all processes running on the system but only show the first 10 lines of the output

ps -ef | tail - list all processes running on the system but only show the last 10 lines of the output

ls | head - list contents of current folder but only show the first 10 lines of the output

ps -ef | grep root - list all processes running on the system but only show those that have root in the filename

ps -ef | grep root | head - list all processes running on the system but only show those that have root in the filename, and only show the first 10 lines of the output

ps -ef | grep root > root_processes.dat - list all processes running on the system but only show those that have root in the filename, and save them to a file called root_process.dat rather than outputting the results to the screen.

ls | head >> contents.dat - append the top 10 rows of the list of contents of current folder to the file contents.dat

ls | head | sort - sort the top 10 rows of the list of contents of current folder

ls | head | sort -r - reverse sort the top 10 rows of the list of contents of current folder

ls -l | grep '^d' - list the sub-folders (sub-directories) in the current folder (directory). I.e. list the files where the first column value of the details for the files starts with a 'd'

ls -l | grep -v '^d' - list all except the sub-folders (sub-directories) in the current folder (directory)

ls -lhS | head - list contents of current directory, including details, and show numbers in human readable format (e.g. 40K), and sort them so the largest are at the top, and only show the first 10 lines of the output

ls -lhS | head -n15 - list contents of current directory, including details, and show numbers in human readable format (e.g. 40K), and sort them so the largest are at the top, and only show the first 15 lines of the output

Job Control

gedit -s - run the job gedit in standalone mode

CTRL-Z - suspend job

CTRL-C - kill job

bg - run job in the background

fg - run job in the foreground (then you could use CTRL-C to kill the job)

jobs - list all current jobs

gedit & - runs gedit job in the bacground

ls > /dev/null - sends output of ls to the null device so you don't have to see it
